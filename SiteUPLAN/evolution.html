Uplan's evolution
Everything you need to know about the application that will change the way you organise your life

We started off this project not knowing much about what a database was, how to create or even use one. We envisioned the ultimate student app, one that would not only help us in our work organisation but our life too. Being a young adult can be overwhelming and it is important that we manage our social life, work life and personal wellbeing, this is the sole purpose of our application.

What the   is a Lite Db Database?

For this application it is crucial to use a database to whom we send information (data) about our tasks, activities and personal choices but also ask for some of that data with a query. We also need the format of the database to be flexible so that we can use any data wether it comes in a music format or just a simple text. LiteDB is a “simple, fast and lightweight embedded .NET document database.”, it's a no SQL database so it's perfectly adaptable for any of our needs. For more info visit:   LiteDb.org

Now that we knew a lot more on how to actually code what we had envisioned, the amount of work and problems weren't as daunting as the first time around. We quite quickly realised that we had trouble with getting the right information for the subtype of any task because the picker wasn't updating its data source. In other words, the picker would always go for the “by default” code and none of the subtype were accurate, to face this issue, we deleted the picker for a few buttons that represented the subtypes and were easily updated. This resolved our AddTask issue! After that, we added a few more functionalities that added subtasks when the work is hard, a secret algorithm that we are not allowed to talk about. For the calendar, we just needed to query for a specific day not a specific date with its hours and minutes (what we were originally doing without noticing it), with that rectified and other issues that came with the page, our calendar was up and running!
